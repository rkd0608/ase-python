# Security Policy

## Supported Versions

The latest `0.1.x` line is the supported release line for security fixes during
this open-source launch phase.

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for security-sensitive reports.

Use one of these private paths:

1. the repository host's private vulnerability reporting flow, if enabled
2. a direct private maintainer contact path listed on the repository profile

Please include:

- affected ASE version or commit
- reproduction steps
- impact assessment
- any proof-of-concept details needed to verify the issue

We will aim to acknowledge valid reports promptly and coordinate disclosure
before any public write-up.
